接口基础知识

1、接口的定义

软件接口：软件程序之间数据交互通道

2、接口分类（类型）

  • 程序内部接口 -- 淘宝系统（登录、商品展示、购物车、下单）
  • 外部接口 -- 快递、支付

3、常见接口（协议）

  • http/https 大多数项目都是采用该协议进行接口的开发 Jmeter/postman
  • Web Service 使用soap协议进行数据传输 soapUI
  • RPC 微服务架构 dubbo thrift

4、接口测试定义

基于某一种协议，客户端发送请求给到服务端，服务端经过内部逻辑处理将响应结果给到客户端，分析响应结果是否符合我们预期（断言）

get与post请求方式的区别？？

1、请求数据位置不同

  • get请求：将请求数据拼接到URL地址的后面，通过附加到问号的后面，而且如果有多组参数的情况下，会通过&符号连接
  • post请求：将请求数据放在请求体里面

2、传输的数据量大小不同

  • get请求数据量大小收到URL地址限制
  • post请求理论上不受到限制，但是实际上还是有限制（浏览器/服务端）

3、安全性不同

  • get请求将请求数据拼接在URL地址后面
  • post请求将请求数据放到请求体，相对而言要安全点

 

接口测试流程

1、需求分析：梳理功能，弄懂业务

2、接口分析：通过接口文档理清接口业务/关联，如果没有接口文档的情况下我们可以通过抓包

3、制定测试计划：人员、进度、时间点

4、测试用例设计，根据接口功能整理提取测试点，等价类、边界值、场景法...

  • 单接口测试场景，保证接口功能正常，处理正常的入参以及得到正常返回，传入异常参数确保服务端正确的返回；需要包含有正常的场景和异常的场景
  • 多接口组合测试场景，站在用户角度考虑业务是否正常

5、执行用例前的准备工作：部署测试环境，准备测试数据

6、测试执行：使用对应接口测试工具发送请求，验证响应结果（Jmeter/postman）

7、BUG管理平台：禅道、bugfree、jira

8、测试报告

Fiddler抓包的原理

核心是通过代理来实现

  • 浏览器使用系统的代理设置
  • Fiddler在启动就会作为系统代理（Tools->Option->Connections->Act as system proxy on startup）
  • 如果要抓https请求包，得需要在Fiddler中安装证书
